H5C3

兼容性

  • 并不是所有的html5都是IE9就兼容的，有一些属性是需要IE10，等更高级的浏览器才能兼容

H5新增标签

  • 六大结构标签
    • header
    • nav
    • section
    • aside
    • article
    • footer

兼容性

  • 新增标签的兼容性解决办法

    • 通过第三方的包解决

    • 通过在内存里面创建标签解决

  • 作用: 增加了语义性, 更有利于SEO优化

  • SEO优化: 含义: 即搜索引擎优化,指通过对网站进行站内优化和修复(网站Web结构调整、网站内容建设、网站代码优化和编码等)和站外优化，从而提高网站的网站关键词排名以及公司产品的曝光度

更多的运用语义化标签, 同时针对一些关键的字符运用权重高的标签去包裹, 如: strong标签去包裹一些重点文字, h1区包裹网站logo

H5新增的智能表单(兼容性较大)

  • H5之前的表单: type = text, submit, reset, radio
  • input表单的type新属性值
    • type=”email” –> 限制用户输入必须为Email类型
    • type=”url” –> 限制用户输入必须为url类型
    • type=”date” –> 自动生成一个日期控件
    • type=”time” –> 同上
    • type=”month” –> 同上
    • type=”week” –> 同上
    • type=”number” –> 限制用户输入必须为数字类型
    • type=”range” –> 产生一个滑动条的表单
    • type=”search” –> 产生一个搜索意义的表单
    • type=”search” –> 生成一个颜色选择表单

智能验证表单(很少使用)

  • required : 验证表单是否为空, 必须配合form表单来使用
  • pattern : 自定义验证表单规则, 匹配正则

注意: 这些属性和自动验证的智能表单必须是submit提交

表单的新属性(一般)

  • placeholder: 占位文本, 体验好(常用)
  • autofocus: 自动获取焦点
  • autocomplete: 提交一次后下次自动补全 (条件:必须提交一次,必须要有name属性(因为用submit提交form表单时input必须要有name属性))

  • multiple: 配合file控件实现多选(重要)

  • form: 配合form表单的id值实现关联, 在任意位置都可以被提交(不建议)

智能感应(较少使用)

  • 利用表单的list=”datalist的id值” 与 datalist这个标签实现联系

H5调用视频(或音频)

  • 标签: video(audio用法一样)
  • 标签属性
    • autoplay => 视频默认加载完播放
    • controls => 播放的控件
    • loop => 控制循环
    • poster =>在视频没有播放的时候的预览图片

总结: 实际工作中,若视频较大建议利用第三方网站去实现; 一些小的广告类的小视频以及宣传类的手机站可用video实现

选择器

  • 关系选择器 ，属性选择器 ，伪类选择器

  • 关系选择器

    • 后代选择器 ul li 选择所有的后代元素
    • 子代选择器 ul > li 选择ul的儿子
    • 紧邻选择器 .box + li 选择.box后面的一个li元素
    • 兄弟选择器 .box ~ li 选择.box后面所有的li元素

  • 属性选择器 ：通过属性来选择对应的元素

    • E[属性名] ： 选择所有的具备这个属性的E元素
    • E[属性名=值] ：选择所有的具备这个‘属性 = 对应值’的E元素

    • E[属性名~=值]：选择所有的包含这个‘属性 = 对应值’的E元素

      css3新增的：

    • E[属性名^=值]：从头去匹配这个‘属性 = 对应值’的E元素
    • E[属性名$=值]：从尾去匹配这个‘属性 = 对应值’的E元素
    • E[属性名*=值]： 从任意位置去匹配这个‘属性 = 对应值’的E元素

  • 伪类选择器(重要)

    • 以前的: before after a：link
    • E：first-child 找到第一个孩子为E的元素 （css2）
    • E：last-child 找到最后一个孩子为E的元素 （css3）

    • E：nth-child（N）找到第N个孩子为E的元素 （css3）

    • 小知识点：
      （1）even 控制偶数 odd控制奇数
      （2）nth-child（5n+1） ： N是从0开始逐次+1
      （3）有一个跟nth-child（）特别类似的一个选择器 nth-of-type（），唯一的区别就是一个侧重的是child 一个侧重的是type类型 后者比前者更宽松
      （4）选择器在PC端保守使用 都是IE9以上兼容（包括9）

  • 其他选择器

    • input:focus{} => 选择获取焦点的表单
      • input:checked => 选择被选中的checkbox表单
    • input:enabled{} => 选择可操控的表单元素
    • input:disabled{} => 选择不可操控的表单
  • 伪元素 before after （重要）
    作用：由css渲染一个虚拟的标签追加到当前元素的内部的前面或者后面单标签是没有伪元素的
  • 总结：
    • 伪元素默认是行内元素，我们可以进行转化，在实际工作中，多用来模拟一些小的标签去装饰
      • 因为伪元素是不存在的，所以不能直接用JS去获取（可以利用类去覆盖之前的样式）
    • 清除浮动的底层原理：就是让一个伪元素（因为伪元素不占dom内存）去清除浮动，从而节约了dom内存，也更加的合理。
      • content一定不能省略，即使里面没有内容，也要写一个空字符串
      • 伪元素推荐用双冒号，但是IE6,7,8不支持双冒号的写法，所以为了兼容考虑，使用单冒号
    • 当伪类需要配合iconfont去使用的话需要打开iconfont.css 拿到content里面对应的值
    • 当伪元素需要hover的效果的时候不能直接hover需要借助于父级 写法：父级：hover：伪元素

  • 值说明:
    • 一个值设置的是盒子的四个角的水平和垂直半径
    • 每个角都可以单独进行设置 其取值顺序是左上 右上 右下 左下顺时针设置
    • 可以简写 简写的逻辑跟padding和margin一样
    • 单位支持像素，和百分比（参照的是宽度和高度）
    • 可以用 水平半径/垂直半径 去单独控制半径 并且每一个半径都可以单独控制

阴影

背景系列

背景图片的基准点

  • 值说明:
    • border-box ：忽略边框 直接从边框的起始 0 ，0点平铺
    • padding-box： 默认值，忽略padding 直接从padding的起始 0 ，0点平铺
    • content-box ：从内容部分开始平铺 跟padding有关系

多重背景

  • 说明：背景图片之间用逗号隔开，可以写多组，最先渲染的图片有可能会遮住后面渲染的图片

控制背景图片的大小(很重要)

  • 值说明:
    • 当只有一个值的情况下，宽度实现拉伸，并且高度会保持等比例，可以支持px，也可以支持百分比，百分比参照的是这个盒子本身的宽高
    • 当有两个值的情况下，宽度和高度会分别拉伸到对应的值，可能会变形，可以支持px，也可以支持百分比，百分比参照的是这个盒子本身的宽高
    • contain 当图片的宽度或者是高度在缩放的时候有一个“碰到”了盒子的边缘，则停止变化，（宽度或高度满足一个即可）
    • 在contain的基础上保证不留白，这就是cover的效果（宽度和高度都需要满足）

盒子内减(很重要)

  • 需要兼容IE9 及其以上 移动端随意
  • 说明: 盒子的实际宽度是 实际宽度 = width + padding + border，而内减属性会自动帮我们 padding 和 border值 ，所以 用了内减的盒子实际宽度就等于width，至于padding和border的值会自动被width内减掉; 盒子继承的是内减过后的宽度

注意:
content-box 不会自动内减
border-box 会自动内减

  • 移动端常用内减配合百分比布局

client等三大家族笔记

offset系列

offsetWidth和offsetHeight

offsetHeight(offsetWidth同理)的构成

  • 包括自身高度, 内边距, 边框, 但是不包括外边距

offsetHeight与style.height的区别

  • demo.style.height只能获取行内样式,若样式写在其他地方则无法获取
  • demo.style.height是字符串(且带单位); 而offsetHeight是数值(重要)
  • domo.style.height可设置行内样式; 而offsetHeight是只读属性,不能设置

offsetLeft与offsetTop

offsetLeft(offsetTop同理)构成

  • 到距离自身最近的(带有定位的)父元素的左侧/顶部的距离,如果所有的父级都没定位则以body为准
  • 即: 自身border左侧到父级padding左侧的距离

offsetLeft与style.left的区别

  • style.left只能获取行内样式
  • offsetLeft只读,style.left可读可写
  • offsetLeft的结果是数值, style.left的结果是字符串且有单位px
  • 最大区别在于offsetLeft以border左上角为基准,style.left以margin左上角为基准

offsetParent

  • 标签.offsetParent表示某个标签外面定位的父盒子(不一定是父节点,也可能是爷爷)
  • 与parentNode区别

注意: offset家族的属性都是只读属性 , 获取的结果是数值

scroll系列

scrollWidth与scrollHeight

  • 是盒子内容的真实宽度和高度, 与盒子大小无关, 只与盒子内容有关

scrollTop与scrollLeft

  • scrollTop是盒子内容被滚动条卷去的头部的高度
  • scrollLeft是盒子内容被滚动条卷去的左侧的高度

注意: scroll家族最常用的地方就是获取页面被卷去的高度和宽度

  • onscroll事件，对于有滚动条的盒子，可以使用onscroll注册滚动事件，每滚动一像素，就会触发该事件。(有兼容性,封装见兼容性文档)

client系列

  • 用于获取盒子可视区的大小
  • 有clientWidth clientHeight clientLeft clientTop
    注意: 如果内容没有超出盒子范围,则clientWidth与scrollWidth相同

clientTop与clientLeft

  • 基本没用, 就是borderTop, borderLeft

onresize

  • 窗口大小被调整时触发

pageX与pageY

用途

  • 在鼠标事件中, 记录鼠标位置信息的属性有很多, 使用最多的还是pageX,pageY, 但是他们有兼容性问题

在现代浏览器中，直接通过事件对象就可以获得pageX与pageY

事件对象

概述

  • 在触发某个事件的时候,都会产生一个事件对象Event, 这个对象中包含所有与事件相关的一些信息, 包括触发事件的元素, 事件的类型以及其他与事件相关的信息

获取事件对象

事件对象的属性

注意: 事件对象的属性有很多,常用的只有鼠标位置信息和键盘码相关信息

  • 记录鼠标位置信息的相关属性
    • screenX与screenY: 光标相对于屏幕左上角的水平位置与垂直位置
    • clientX与clientY: 光标相对于可视区左上角的水平位置与垂直位置
    • pageX与pageY: 光标相对于网页(文档document)左上角的水平位置与垂直位置
  • 记录键盘码的属性
    • event.keyCode: 键盘按下那个键的键盘码

事件的三个阶段

  • 事件的捕获阶段
  • 事件的目标阶段
  • 事件的冒泡阶段

事件冒泡

  • 当一个元素的事件被触发时, 同样的事件将会在该元素的所有祖先元素中依次触发(IE提出)

阻止事件冒泡

  • 正常浏览器

  • ie678

事件捕获

  • 事件捕获是火狐浏览器提出来的，IE678不支持事件捕获（基本上，我们都是用事件冒泡） 事件的处理将从DOM层次的根开始，而不是从触发事件的目标元素开始，事件被从目标元素的所有祖先元素依次往下传递

量词

量词用来控制出现的次数，一般来说量词和边界会一起使用

  1. *表示能够出现0次或者更多次，x>=0;
  2. +表示能够出现1次或者多次，x>=1
  3. ?表示能够出现0次或者1次，x=0或者x=1
  4. {n}表示能够出现n次
  5. {n,}表示能够出现n次或者n次以上
  6. {n,m}表示能够出现n-m次

  • 标签.currentStyle

分支学习

分支

问题：

多任务并行开发的时候，避免不了，各个任务之间相互影响，我们也可以采用备份文件夹的方式，来避免影响，但是这种方式不可取，因为一旦任务过多，文件夹数量也会随之增多不容易维护，最终的代码合并，也会出现问题

分支

git中有个内容，叫做分支，分支的作用，其实和刚才我们做的备份一个文件夹的功能一致，就是对于当前的代码，做一个备份

分支的使用

  1. 创建分支

  2. 切换分支

  3. 合并分支

  4. 删除分支

  5. 创建并切换分支

冲突问题

分支A中创建了文件xx.txt, 分支B中也创建了个xx.txt，那么git就会发现，有两个同名的文件，这时候，就是冲突产生了，冲突产生之后，合并会失败，需要先解决冲突，然后再次提交，才能正常的合并。

解决冲突

冲突产生之后，git会告知哪个文件冲突了，然后在文件中，会用 >>>> ==== <<< 这样的方式将冲突的内容标记出来，我们只需要将标记删除，然后将代码改成自己想要的结果，再次add commit即可！！

分支的使用原则

只要有新的任务，就开辟一个新的分支去做，做完之后，将分支合并回去就可以了！！

master分支

这个分支，是git默认自动创建的！！

创建的时机，并不是在git init之后就会有，而是，在第一次提交完成之后，才会默认的创建出来这个分支！！

分支切换要注意的事项

必须先将所有的更改，做一次提交之后，再切换分支！！！

回滚操作的恢复操作

  1. 查看所有的操作日志

  2. 创建新的分支，将要恢复到的版本号作为最后一个参数
