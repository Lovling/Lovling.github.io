DOM(文档对象模型)

DOM样式操作

别忘记先获取元素

标签样式的设置方式

  • style
    设置的样式值为字符串类型, 有单位加单位 , 且设置的样式为行内样式

  • getComputedStyle()

  1. 标准方法,获取的是计算过后的样式(即获取的样式值是标签最终生效的那个样式值)
  2. 获取的结果为对象形式, 是当前标签所有样式的集合

    注意: 兼容性问题可查看兼容性文档或者MDN解决;
  • 标签.currentStyle
    该标签是IE自身支持一种样式获取的方式(所以有兼容性)

标签内容的设置方式

  • 使用innerHTML属性设置: 可设置内部的纯文本和标签
  • 使用innerText属性设置: 可以针对纯文本的设置操作使用
  • 使用textContent属性设置: 纯文本的设置操作

注意:

  1. 后两个有兼容性问题,可查看兼容性文档或者MDN解决;
  2. 该方法只能进行样式值的获取, 结果只读不可设置(结果为字符串带单位)

DOM事件

组成部分:

  • 事件源(触发事件的元素)(被动)
  • 事件类型(点击, 鼠标移入移出等)
  • 事件处理程序(函数类型)

标签内自带属性的操作方式

对于标签行内自身支持的行内属性 , 可以通过标签.属性名的方式直接操作
注意: 在设置时, 由于标签的class属性是一个关键字 , 所以改为className

标签内自定义属性的操作方式(重要)

获取行内自定义属性值(也可以获取自带属性,不常用)

标签.getAttribute();

设置行内自定义属性值

标签.setAttribute(“要修改的属性名”,”修改成什么值”) 没有返回值

移除自定义属性

标签.removeAttribute(“要移除的属性名”)

索引的取值

  • 自定义属性来操作

  • 自调用函数来操作

  • 最常用的一种方式(也是通过自定义属性)

阻止a标签默认跳转的方式

如果给a标签设置点击事件, 那么自定义点击事件会先执行, 而a标签自带的跳转效果会后执行
阻止默认事件:

  1. 在自定义事件代码的最后添加 return false;(跳出函数)
  2. e.preventDefault()

DOM节点与节点树

节点

定义: 对节点树中的所有形式内容的统称

节点种类

  • 元素节点: 标签, 页面元素
  • 文本节点: 文本内容
  • 属性节点: 标签的行内属性
  • 注释节点: 注释
  • 根节点: document(DOM中的顶级对象, 所有DOM的内容均为它的下属内容)

节点树

好处: 可以让元素节点的访问方式变得更加简单

克隆节点

  • 需要克隆的标签不要设置id名

动态数组与静态数组的问题

动态创建元素

作用

  • 提高页面的响应速度(动态加载结构)
  • 节省流量(动态加载图片)

方式

  • document.write

  • innerHTML

  • document.createElement()

  • 三者的使用场景

  1. 如果需要创建的结构大量重复且结构比较复杂时,推荐使用innerHTML(操作方式为字符串拼接)
  2. 如果要进行动态创建元素的位置上具有旧标签, 并且旧标签上可能具有事件,必须使用document.createElement

indexOf

  • 匹配成功返回索引值,匹配失败返回-1
  • 使用indexOf想要匹配成功,参数字符串必须比调用的字符串短
  • 使用空字符串对其他字符串进行匹配时, 可以在开头立刻匹配成功
  • 还有第二个参数不常用: 表示调用该方法的字符串中开始查找的位置, 可以是任意整数

BOM笔记

BOM(browser object model)

  • 浏览器对象模型
  • 作用: 提供了一套可以操作浏览器相关功能的方法
  • DOM实际上是BOM中的一个重要组成部分

BOM顶级对象window

  • window也是js在浏览器环境中的顶级对象
  • 由于是顶级对象,所以属性和方法很多, 当我们使用这些方法时,可以不加window
  • 所有的全局变量以及函数都是window的属性和方法

注意: 如果想要在局部作用域中访问全局变量的值,推荐使用window.属性名的形式

location对象

window.location

页面跳转

  • location.href = “要跳转到的地址”
  • location.assign(“要跳转的地址”) 与href相同
  • location.replace(“要跳转的地址”) 没有回退的记录

页面刷新

  • location.reload()

其他

  • window.location.hash
  • window.location.host
  • window.location.hostname
  • window.location.port
  • window.location.pathname
  • window.location.protocol
  • window.location.search

navigator对象

window.navigator

  • navigator.platform 用于显示当前设置的运行平台(系统)
  • navigator.userAgent 用户代理字符串: 老式的兼容性检测方式(用于判断浏览器的信息)

history方法

  • history.go(1) 前进页面
  • history.go(-1) 后退页面
  • history.forward() 前进页面
  • history.back() 后退页面

其他方法

  • open打开一个新页面

  • close关闭页面 window.close()

  • 其他小方法: moveTo moveBy resizeTo resizeBy

定时器(重点)

setTimeout

  • 参数1: 时间到达要做的事情
  • 参数2: 延迟执行的时间(毫秒形式)
  • 返回值: 为当前定时器的标识(定时器id) 常声明为timer
  • timeout的清除: clearTimeout(timer) 参数是定时器的标识

setInterval

  • 间隔执行
  • 参数同timeout
  • 清除方式: clearInterval(timer)

远程仓库

git

git 是一个分布式的 版本控制工具，特点，就是版本库存放在本地！！！

远程仓库的作用

就是共享版本库！！！ 共享代码！！！

远程仓库的创建

远程仓库其实就是一个裸仓库

裸仓库

创建裸仓库

裸仓库创建完成之后，就不需要做任何操作了，在裸仓库中不能进行任何的开发操作

远程仓库的操作命令

  1. 给远程仓库推送内容

  2. 获取远程仓库中所有的内容

  3. 获取远程仓库中更新了的内容

注意

只有在远程仓库中从未推送过任何内容的时候，我们才能随意向里面推送任何内容

但是，一旦已经推送过一次内容了，那么之后就不能随便的向远程仓库推送内容了，能推送的，只能是和第一次推送相关的修改的内容！！

每次在向远程仓库中推送新内容的时候，必须先进行一次获取更新内容的操作（pull），否则可能推送失败！！！

分支学习

分支

问题：

多任务并行开发的时候，避免不了，各个任务之间相互影响，我们也可以采用备份文件夹的方式，来避免影响，但是这种方式不可取，因为一旦任务过多，文件夹数量也会随之增多不容易维护，最终的代码合并，也会出现问题

分支

git中有个内容，叫做分支，分支的作用，其实和刚才我们做的备份一个文件夹的功能一致，就是对于当前的代码，做一个备份

分支的使用

  1. 创建分支

  2. 切换分支

  3. 合并分支

  4. 删除分支

  5. 创建并切换分支

冲突问题

分支A中创建了文件xx.txt, 分支B中也创建了个xx.txt，那么git就会发现，有两个同名的文件，这时候，就是冲突产生了，冲突产生之后，合并会失败，需要先解决冲突，然后再次提交，才能正常的合并。

解决冲突

冲突产生之后，git会告知哪个文件冲突了，然后在文件中，会用 >>>> ==== <<< 这样的方式将冲突的内容标记出来，我们只需要将标记删除，然后将代码改成自己想要的结果，再次add commit即可！！

分支的使用原则

只要有新的任务，就开辟一个新的分支去做，做完之后，将分支合并回去就可以了！！

master分支

这个分支，是git默认自动创建的！！

创建的时机，并不是在git init之后就会有，而是，在第一次提交完成之后，才会默认的创建出来这个分支！！

分支切换要注意的事项

必须先将所有的更改，做一次提交之后，再切换分支！！！

回滚操作的恢复操作

  1. 查看所有的操作日志

  2. 创建新的分支，将要恢复到的版本号作为最后一个参数
